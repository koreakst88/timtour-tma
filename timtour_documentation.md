# TimTour TMA — Проектная документация

> **Статус:** Мозговой штурм завершён, документация согласована  
> **Стек:** Next.js 14 · Tailwind CSS · Shadcn UI · Supabase · Vercel · Telegram Bot API  
> **Аудитория:** Русскоязычные пользователи в Южной Корее  
> **Модель работы:** Заявка → уведомление менеджеру → менеджер связывается с клиентом

---

## О проекте

TimTour — туристическое агентство в Южной Корее (@timtour22, 22К подписчиков в Instagram). Организует туры внутри Кореи и по странам ЮВА (Вьетнам, Таиланд, Фукуок и др.).

TMA (Telegram Mini App) — основная точка контакта с клиентом. Пользователь попадает в приложение через QR-код, ссылку или кнопку в Telegram-канале. Без регистрации — данные подтягиваются из Telegram автоматически.


## Аналитика (Amplitude)

Подключается сразу на этапе прототипа.

| Событие | Триггер |
|---------|---------|
| `app_opened` | Открыл TMA |
| `country_selected` | Выбрал страну направления |
| `tour_viewed` | Открыл карточку тура |
| `tour_favorited` | Добавил тур в избранное |
| `booking_started` | Нажал "Забронировать" |
| `booking_completed` | Отправил заявку |

---

## Передача клиенту

Разработка ведётся на боте разработчика. При передаче клиенту:
1. Клиент создаёт бота через @BotFather → получает токен
2. Меняется одна переменная `TELEGRAM_BOT_TOKEN` в Vercel
3. Supabase переносится в аккаунт клиента или остаётся на хостинге разработчика
4. Клиент полностью самостоятелен — управляет контентом через admin-панель без программиста

## Шаг 1 — Инфраструктура и базовая настройка

**Что делаем:**
- Создаём репозиторий на GitHub
- Инициализируем проект Next.js 14 с Tailwind CSS и Shadcn UI
- Подключаем Supabase — создаём проект, настраиваем таблицы (tours, bookings, users)
- Настраиваем Supabase Storage — бакеты для фото и видео туров
- Создаём Telegram-бота, прописываем переменные окружения
- Деплой заготовки на Vercel
- Подключаем Amplitude — базовая инициализация

**Таблицы Supabase:**

```
countries
  id, name, flag_emoji, cover_url, is_priority, order, is_active

tours
  id, country_id, title, description,
  price, duration_days, type (group|individual),
  is_active, created_at

tour_media
  id, tour_id, url, type (photo|video), order

tour_dates (для групповых туров)
  id, tour_id, date_start, date_end, seats_total, seats_left

bookings
  id, tour_id, user_tg_id, user_name, phone,
  comment, travel_date, people_count,
  status (new|processing|confirmed), created_at

favorites
  id, user_tg_id, tour_id, created_at

reviews
  id, tour_id, user_tg_id, user_name, text,
  rating, is_visible, created_at
```

**Результат шага:** Проект запущен на Vercel, база данных настроена, бот отвечает на /start
